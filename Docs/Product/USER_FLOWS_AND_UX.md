# User Flows and UX
## Jacked into the Matrix

## UX Principles

- keep the phone rich and comfortable
- keep the glasses minimal and glanceable
- always show the next best instruction
- never bury critical safety information

## Top-Level Navigation

- Home
- Categories
- Favorites
- Recents
- Glasses
- Settings

## Core Flow 1: Run a Saved Script

1. User opens the app
2. User taps a category or recent script
3. User reviews summary, risk level, warnings, tools, and steps
4. User taps **Send to Glasses**
5. G2 shows step 1
6. User advances next/back using phone or supported input
7. Progress is saved automatically

## Core Flow 2: Resume a Script

1. User opens the app
2. Home shows **Continue where you left off**
3. User reopens the active script
4. Playback resumes at the last saved step

## Core Flow 3: Drill Mode

1. User opens a study-friendly script
2. User selects **Drill Mode**
3. App shows prompt or checkpoint
4. User taps to reveal next answer or step
5. Progress continues until finished

## Core Flow 4: Favorite a Script

1. User opens a script
2. User taps favorite
3. Script appears in Favorites
4. Favorite syncs to Home surfacing logic

## Key Screens

### Home
Sections:
- Continue where you left off
- Favorites
- Categories preview
- Recent scripts

### Category Screen
Shows category cards and script lists.

### Script Detail
Shows:
- title
- summary
- risk level
- warnings
- tools / ingredients / materials
- prerequisites
- numbered steps
- verification checklist
- favorite button
- send to glasses button

### Playback Screen
Shows:
- current step
- tip line
- progress
- previous / next controls
- mode switcher

### Glasses Status
Shows:
- connection state
- active script
- last synced time
- resend / clear actions

## Glasses UX Rules

- one dominant instruction at a time
- optional one-line tip
- visible progress indicator
- no long paragraphs
- no multi-panel complexity

## Example G2 Card Format

Title: `OPENCLAW UPDATE`
Progress: `3/7`
Step: `Run the update command`
Tip: `Confirm you are in the right repo`
